# Claude 협의 기반 지식 관리 시스템 템플릿 v0.1.0

## 🎯 개요

이 템플릿은 Claude와 사용자 간의 효과적인 협업을 위한 **수동 지식 관리** 가이드입니다.

**핵심 철학**: 완벽한 자동화보다 실용적인 협력
**실제 동작**: 파일 기반 컨텍스트 공유, 수동 패턴 관리, 명시적 참조

---

## 🏗️ 계층적 시냅스 아키텍처

### 📊 단순 파일 구조
```
CLAUDE.md ← 프로젝트 가이드 (수동 참조)
    ├─ patterns/ (성공/실패 사례 기록)
    └─ archive/ (오래된 문서 보관)
```

### 🎯 실제 작동 방식
- **CLAUDE.md**: 프로젝트별 작업 가이드 (매 세션 읽기 필요)
- **patterns/**: 유용한 패턴 수동 기록 및 참조
- **archive/**: 더 이상 사용하지 않는 문서 보관

## 🔄 스마트 트리거 시스템 (고도화)

### 자동 작업 유형 감지
```markdown
# 연쇄 발견 패턴 (Chain Discovery)
트리거: "분석해줘", "찾아봐", "확인해줘", "상태 체크", "문제 진단"
→ 선택적 로딩: 현재 프로젝트 시냅스 + 관련 도구 시냅스

# 계획적 작업 패턴 (Planned Execution)
트리거: "만들어줘", "구현해줘", "설계해줘", "작성해줘"
→ 선택적 로딩: 성공 패턴 + 작업 방법론 + 실패 교훈

# 컨텍스트 로딩 작업
트리거: "시냅스 활성화해줘", "컨텍스트 로딩", "패턴 연결"
→ 선택적 로딩: 전체 시냅스 네트워크 + 협의 결정사항
```

### 🎯 컨텍스트 우선순위 시스템
```markdown
🔥 HOT - 즉시 접근 컨텍스트
- 현재 진행 중인 작업
- 핵심 시스템 상태  
- 최근 변경 사항

🟡 WARM - 정기 참조 컨텍스트
- 프로젝트 구조 및 설정
- 도구별 사용 패턴
- 작업 방법론

❄️ COLD - 아카이브 컨텍스트
- 버전 히스토리
- 실패 사례 및 교훈
- 레거시 정보
```

### 컨텍스트 선택적 로딩 전략
- **프로젝트 키워드**: 도메인별 핵심 용어 → 해당 도구 시냅스 선택적 로딩
- **문제 해결 키워드**: 오류, 실패, 디버깅 → lessons_learned.md 우선 로딩
- **최적화 키워드**: 개선, 효율성, 성능 → successful_approaches.md 우선 로딩

## 🎯 작업 방법론 트리거 (구체적 키워드)

### 🚨 연쇄 발견 작업 감지 신호
- **전면 변경 패턴**: "모든 [대상]을 [새값]으로", "전체적으로 [변경사항]"
- **일관성 키워드**: "[용어] 통일", "[구조] 리팩토링", "[방식] 표준화"
- **범위 확장 신호**: "관련된 모든", "연관된 부분도", "일관성을 위해"
- **실제 예시**:
  - "모든 로그를 새 형식으로 변경"
  - "전체 API 응답 구조 리팩토링"
  - "용어를 manager에서 coordinator로 통일"
→ `ai_docs/patterns/work_methodology.md` 참조 (순차 처리 필수)

### 📋 계획적 작업 감지 신호
- **명확한 목록**: "다음 N개 파일에서", "[파일1], [파일2], [파일3]에서"
- **구체적 매핑**: "[파일1]에서 A를 B로, [파일2]에서 C를 D로"
- **체크리스트 제시**: "1. [작업1] 2. [작업2] 3. [작업3]"
- **실제 예시**:
  - "config.js, utils.js, main.js에서 port 변수를 3000으로"
  - "5개 컴포넌트의 import 문 정리"
→ MultiEdit 등 배치 처리 도구 활용 고려

### ⚠️ 주의사항 참조 신호
- **효율성 관련**: "효율화", "최적화", "자동화", "개선 방안"
- **성능 관련**: "더 빠르게", "배치 처리", "일괄 작업"
- **복잡성 관련**: "복잡한 시스템", "고도화", "완벽한 자동화"
→ `ai_docs/patterns/lessons_learned.md` 먼저 확인

### 3. 🔍 작업 유형 판별

#### 연쇄 발견 작업 (Chain Discovery Work)
- **특징**: 수정 범위가 점진적으로 확장
- **접근**: 순차적 처리, 배치 처리 금지
- **예시**: 용어 통일, 구조 리팩토링

#### 계획적 일괄 작업
- **특징**: 명확한 범위와 목록
- **접근**: 배치 도구 활용 가능
- **예시**: 다수 파일의 동일 패턴 수정

### 4. 📝 패턴 등록 양식

#### successful_approaches.md
```markdown

### 💡 협의 과정: [단계별 협의 내용]

### 🎯 핵심 성공 요인: [성공 이유]

```

이 파일을 읽는 순간 허브 로딩 + 컨텍스트 활성화 옵션 제시

🧠 [프로젝트] 허브 로딩 완료

💬 "시냅스 활성화해줘"라고 하시면

## 🤝 협의 프로토콜
[위 협의 프로토콜 템플릿 복사]

## 🎯 컨텍스트 우선순위
🔥 HOT: [현재 핵심 작업]
🟡 WARM: [정기 참조 항목]  
❄️ COLD: [아카이브 정보]
